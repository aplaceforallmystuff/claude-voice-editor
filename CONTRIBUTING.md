# Contributing to Claude Voice Editor

Thanks for your interest in improving the voice-editor skill!

## Ways to Contribute

### Improve Editing Passes

The 6-pass system can be refined:

- Better questions to ask at each pass
- More specific actions to take
- Additional patterns to look for

### Expand Problem Patterns

Help identify more common problems:

- New symptoms to detect
- Better fixes for existing issues
- Industry-specific patterns

### Add Editing Examples

Real-world examples are valuable:

- Before/after comparisons
- Different content types
- Various voice styles

### Enhance Integration

Improve workflow with other skills:

- Better handoff points
- Shared patterns
- Combined workflows

## Submitting Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b improve-rhythm-pass`)
3. Make your changes to `skills/voice-editor/SKILL.md`
4. Test with various content types
5. Submit a pull request

## Guidelines

### Keep It Practical

Editing guidance should be:
- Actionable (not just descriptive)
- Quick to apply (each pass 5-10 minutes)
- Verifiable (can check if done)

### Maintain the 6-Pass Structure

The passes have a specific sequence:
1. Structure first (organization)
2. Then voice (personality)
3. Then cleanup (slop removal)
4. Then depth (specificity)
5. Then polish (rhythm)
6. Finally verify (authenticity)

Changes should respect this flow.

### Focus on Voice, Not Grammar

This skill is about voice matching, not:
- Grammar correction
- Spelling fixes
- Punctuation rules

Those belong in other tools.

## Questions?

Open an issue for discussion before major changes.
